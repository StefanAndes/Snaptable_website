Security & Privacy FAQ

Is my data secure?
Absolutely. SnapTable uses military-grade security:
  • Zero storage: Screenshots never touch your hard drive
  • Bank-level encryption: All data encrypted in transit with TLS 1.2+
  • No retention: Images are deleted immediately after processing
  • Enterprise cloud: Powered by AWS with 99.99% uptime SLA

How it works:

Screenshot ‚Üí Encrypted transmission ‚Üí AWS processing ‚Üí Result to clipboard ‚Üí Screenshot deleted

Total time: 2-5 seconds. Storage: RAM only. Persistence: Zero.
Can you see my screenshots?
No. We never see your data:
  • Screenshots go directly from your computer to AWS Textract (not our servers)
  • Processing is fully automated - no human review
  • AWS employees cannot access customer data without explicit permission
  • We only see license validation (key + machine ID - no content)
Where is my data stored?
It's not. Your screenshots are never stored:
  • In RAM only: Screenshot exists in memory for ~2-5 seconds
  • No temp files: Nothing written to disk
  • No cache: No history of captures
  • No cloud storage: AWS processes and immediately discards images
  • No backups: Screenshots cannot be recovered (even by us)
Is this HIPAA compliant for medical records?
Yes! SnapTable is HIPAA-eligible:
  • AWS Textract is HIPAA-eligible service
  • Business Associate Agreement (BAA) available
  • Audit logging via AWS CloudTrail
  • Complies with ePHI security requirements
Healthcare organizations: Contact us for BAA setup. Takes 1-2 business days, no extra cost.
Can I use this for financial documents?
Absolutely. Perfect for finance teams:
  • PCI DSS Level 1 certified (highest payment card security)
  • SOC 2 Type II compliant (independent security audit)
  • Ideal for: invoices, receipts, bank statements, tax forms
  • Used by accounting firms and financial institutions
Note: Do not capture full credit card numbers directly (PCI DSS requirement). Redact PANs before capture.
What about AI training on my data?
Opted out. Your data is never used for AI training:
  • AWS AI opt-out configured by default
  • Your screenshots never train machine learning models
  • Covered by AWS Customer Agreement
  • Images deleted immediately after processing

Can I use this offline?
No - internet required. SnapTable needs cloud connectivity for:
  • AWS Textract API (advanced OCR processing)
  • License validation
Why cloud processing?
AWS Textract provides 99%+ accuracy - far better than local OCR. The trade-off is internet requirement, but we maximize security with encryption and zero storage.
What if sensitive data is accidentally captured?
It's already gone. No recovery needed:
  1. Screenshot deleted from RAM within seconds
  2. AWS Textract discarded the image immediately after processing
  3. Only the text result remains in your clipboard (clear by copying anything)
  4. No backups, no logs, no recovery possible
Unlike email/cloud storage where you must request deletion, SnapTable data self-destructs automatically.
Is my license key secure?
Yes. License keys are protected:
  • Stored locally in config file (your Windows account only)
  • Never exposed in logs or error messages
  • Transmitted only for validation (encrypted)
  • Cannot access AWS or your data (separate credentials)
How do I completely delete my data?
Your data is already deleted! Screenshots are never stored.
To reset the app (remove license/usage data):
  1. Close SnapTable
  2. Delete: snaptable_config.json
  3. Delete: snaptable_usage.json
  4. Delete: snaptable.log
These files only contain license info and usage counts - never screenshot content.
